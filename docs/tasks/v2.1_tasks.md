### T3: GIF生成流程修复 (P0 - 高优先级)

**负责人**: `backend-developer`, `ai-developer`  
**预估工时**: 4h  
**依赖**: 无

#### 问题描述
用户反馈上传图片后，`generate_all_emotions()` 应该生成3种情绪GIF（happy/sad/excited），但实际不工作。

#### 代码流程分析
```
用户上传图片
    │
    ▼
POST /avatars/upload (avatar.py:169)
    │
    ▼
process_avatar_task() (avatar.py:54)
    │
    ├── 上传参考图到 imgbb 或 base64
    ├── 上传人物照片到 imgbb 或 base64
    │
    ▼
generate_all_emotions() (emotion_animator.py:134)
    │
    ├── 对每个情绪调用 generate_emotion_frames()
    │   │
    │   ▼
    │   调用火山引擎 API (doubao-seedream-4-5-251128)
    │   │
    │   ▼
    │   返回帧序列 → 移除绿幕 → 生成GIF
    │
    ▼
保存到 static/avatars/{task_id}/
```

#### 可能的问题点
1. **API Key问题**: `ARK_API_KEY` 或 `IMGBB_API_KEY` 未配置
2. **API响应异常**: 火山引擎图像生成API可能返回错误
3. **图片上传失败**: imgbb上传可能失败，base64 fallback可能有问题
4. **错误处理不足**: 异常被静默捕获，用户无法得知失败原因
5. **流式响应处理**: `stream=True` 的响应处理可能有问题

#### 子任务
| 子任务ID | 描述 | 负责人 | 依赖 |

| T3.1 | 问题排查：添加详细日志，定位失败点 | ai-developer | - |
| T3.2 | 修复API调用或错误处理 | backend-developer | T3.1 |
| T3.3 | 添加重试机制和更好的错误提示 | backend-developer | T3.2 |
| T3.4 | 编写单元测试 | backend-developer | T3.3 |

##### T5.1 数据库模型 (database-developer, 1.5h)

```python
class TreeHolePost(Base):
    __tablename__ = "tree_hole_posts"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(String, nullable=True)  # 允许匿名
    emotion = Column(String)  # joy, sad, hope, anxiety
    content = Column(Text)
    is_anonymous = Column(Boolean, default=False)
    likes_count = Column(Integer, default=0)
    created_at = Column(DateTime, default=datetime.utcnow)

class TreeHoleLike(Base):
    __tablename__ = "tree_hole_likes"
    
    id = Column(Integer, primary_key=True)
    post_id = Column(Integer, ForeignKey("tree_hole_posts.id"))
    user_id = Column(String)
    created_at = Column(DateTime, default=datetime.utcnow)
```
