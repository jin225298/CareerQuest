# 产品需求分析报告

## 1. 项目概述

### 1.1 核心目标
构建一个**游戏化求职AI平台**，通过像素风格数字分身 + AI面试模拟 + 成长系统，帮助求职者提升面试能力、优化求职流程，让求职过程变得有趣且有成就感。

### 1.2 目标用户
| 用户群体 | 特征 | 核心诉求 |
|---------|------|---------|
| 应届毕业生 | 缺乏面试经验，求职焦虑 | 模拟练习、指导陪跑 |
| 职场转型者 | 需要针对性准备 | 岗位分析、策略规划 |
| 考公考编人群 | 面试形式固定 | 高频题目练习、结构化训练 |
| 海外求职者 | 时差、语言障碍 | 异步面试、多语言支持 |

### 1.3 使用场景
- **日常练习**：利用碎片时间进行模拟面试
- **面试冲刺**：收到面试通知后的定向准备
- **复盘沉淀**：面试后的回顾与改进
- **求职规划**：岗位筛选、投递策略制定

### 1.4 当前痛点
1. **练习枯燥**：传统面试准备缺乏趣味性，难以坚持
2. **反馈缺失**：独自练习无法获得有效反馈
3. **焦虑无处释放**：求职压力无处倾诉和缓解
4. **信息不对称**：岗位JD、面试问题难以解读
5. **简历优化难**：不知道如何针对性优化

### 1.5 成功标准
- 用户完成至少3次完整模拟面试
- 周留存率 > 40%
- 用户面试能力评分提升 > 20%
- NPS（净推荐值）> 50

### 1.6 隐含需求
- 社交分享：炫耀成就、像素小人
- 数据可视化：进度追踪、能力雷达图
- 个性化：不同行业的定制化体验
- 情感连接：对像素分身的情感投入

## 2. 竞品分析

### 2.1 参考产品：NYX（AI版拓麻歌子）

**核心借鉴点**：
| 维度 | NYX方案 | 本项目借鉴 |
|------|---------|-----------|
| 像素动画生成 | 上传照片 → AI生成动画分身 | 同样方案，应用于求职场景 |
| 数值映射 | 精神/健康值 + 状态动画 | 武力/心情/HP + 面试状态 |
| 语音交互 | 自然语言记录行为 | 面试对话、Lip Sync同步 |
| 游戏化 | 任务、成就、记忆 | 每日任务、成就系统、面试记录 |
| 技术栈 | React + Python | 可复用，降低学习成本 |
| 成本控制 | 像素动画 0.2元/9秒 | 面试场景动画成本可控 |

**差异化方向**：
- NYX：健康生活记录 → 本项目：求职能力成长
- NYX：自我养育 → 本项目：职业能力养成
- NYX：单角色 → 本项目：多NPC角色系统

### 2.2 市场竞品概览

| 产品类型 | 代表产品 | 优势 | 劣势 |
|---------|---------|------|------|
| 面试AI工具 | Final Round AI、Verve AI | 实时辅助、录音分析 | 缺乏趣味性、价格昂贵 |
| 求职平台 | Boss直聘、拉勾 | 岗位丰富、流量大 | 无面试训练功能 |
| AI简历工具 | Kickresume、Rezi | 简历优化、模板丰富 | 缺乏面试准备闭环 |
| 游戏化学习 | Duolingo、多邻国 | 成瘾性强、激励有效 | 非求职垂直领域 |

**竞争策略**：
- **差异化**：像素风格 + 游戏化 + 多NPC角色
- **切入点**：年轻人市场（95后、00后对像素风接受度高）
- **护城河**：用户数据沉淀（面试记录 → 个性化推荐）

## 3. 功能需求

### 3.1 功能可行性评估

#### 🔲 P0 核心功能

| 功能模块 | 可行性 | 说明 |
|---------|--------|------|
| 像素小人生成系统 | ✅ 可行 | NYX已验证，技术路径清晰 |
| 面试语音交互系统 | ✅ 可行 | ASR + TTS + LLM 成熟 |
| Lip Sync嘴型同步 | ✅ 可行 | Web Audio API + 时间轴控制 |
| 面试评分系统 | ✅ 可行 | LLM多维度评估，输出结构化数据 |
| 问卷调查系统 | ✅ 可行 | 常规表单功能 |
| 基础动画状态 | ✅ 可行 | 呼吸、说话、聆听等通用动画 |

#### 🔲 P1 增强功能

| 功能模块 | 可行性 | 说明 |
|---------|--------|------|
| 多NPC角色系统 | ⚠️ 有风险 | 需设计每个角色的人设、话术库，工作量较大 |
| 实时援护系统 | ⚠️ 有风险 | 需实时分析用户回答，判断"卡壳"时机 |
| 压力槽可视化 | ✅ 可行 | 基于语音/回答质量实时计算 |
| 每日任务系统 | ✅ 可行 | 任务设计需结合求职场景 |
| 简历生成系统 | ⚠️ 有风险 | 需要大量用户数据输入，冷启动困难 |
| 岗位推荐系统 | ⚠️ 有风险 | 依赖第三方API或爬虫，数据质量难保证 |

#### 🔲 P2 远期功能

| 功能模块 | 可行性 | 说明 |
|---------|--------|------|
| 法官大人（岗位避坑） | ❓ 待确认 | 需要大量真实岗位数据、员工评价数据源 |
| 职场翻译官（JD解码） | ✅ 可行 | LLM可处理，但需构建"黑话词典" |
| 外号员（求职科普） | ✅ 可行 | 内容型功能，需持续更新知识库 |
| 社交分享功能 | ✅ 可行 | 生成分享图，接入社交平台SDK |
| 多人竞技/排位 | ❌ 不建议 | 初期用户量少，匹配困难，建议后期迭代 |

### 3.2 功能详细拆解

#### 3.2.1 像素小人生成系统（P0）

**输入**：
- 用户上传照片（可选）
- 问卷调查结果（职业方向、性格、身份标签）

**输出**：
- 3x3精灵图
- 基础动画组：呼吸、说话、聆听、点头、流汗、发抖

**生成逻辑**：
```
问卷标签 → 模板选择 → 配色方案 → 装饰元素 → 最终精灵图
```

**示例**：
- 软件开发：眼镜 + 笔记本 + 咖啡杯
- 算法岗：矩阵背景 + 数学符号
- 考公考编：正装 + 文件夹

#### 3.2.2 面试系统核心流程（P0）

```
[开始面试]
    ↓
[选择面试类型]：技术面/HR面/行为面/模拟群面
    ↓
[面试官NPC出场]：根据岗位类型匹配
    ↓
[语音交互循环]
    用户说话 → 小人说话动画 + Lip Sync
    AI提问 → 小人聆听动画
    ↓
[实时援护]（可选开启）：卡壳时陪跑员提示
    ↓
[面试结束]
    ↓
[多维度评分]：表达能力/逻辑思维/专业能力/应变能力/情绪管理
    ↓
[数值更新]：武力值+/心情值波动/成就解锁
    ↓
[复盘记录]：存入Memory模块
```

#### 3.2.3 角色属性系统（P0）

| 属性 | 范围 | 影响因素 | 影响 |
|------|------|---------|------|
| 武力值（Power） | 0-100 | 面试表现、任务完成 | 解锁高难度面试、NPC态度 |
| 心情值（Mood） | 0-100 | 面试结果、连续登录 | 动画表情、暴击率 |
| 健康值（HP） | 0-100 | 任务负荷、休息 | 动画状态、可用功能 |
| 胜场（Wins） | 累计 | 面试通过次数 | 成就解锁、排行榜 |
| 记忆（Memory） | 列表 | 所有面试记录 | 复盘、个性化推荐 |

#### 3.2.4 NPC角色系统（P1）

| NPC角色 | 职能 | 人设风格 | 交互触发 |
|---------|------|---------|---------|
| 面试官 | 提问、追问、评分 | 严肃/友善/压力面（多种人设） | 进入面试 |
| 陪跑员 | 安抚、提示、鼓励 | 温暖、有同理心 | 面试中、面试后 |
| 法官大人 | 判断岗位是否有坑 | 公正、犀利 | 岗位分析 |
| 外号员 | 科普求职知识 | 幽默、话痨 | 主动咨询 |
| 职场翻译官 | 解码JD黑话 | 专业、简洁 | 粘贴JD |

## 4. 创新性建议

### 4.1 游戏化创新

| 创新点 | 优先级 | 说明 |
|--------|--------|------|
| **面试副本系统** | P1 | 不同难度副本：新手村（基础行为面）→ 进阶副本（技术面）→ 地狱副本（压力面） |
| **装备系统** | P1 | 解锁像素装扮：完成X次面试解锁"面试达人"徽章、连胜解锁特效边框 |
| **心情日记** | P2 | 小人自动记录面试情绪，生成"求职心情曲线" |
| **队友系统** | P2 | 好友互看像素小人、互相点赞、组队练习 |

### 4.2 交互创新

| 创新点 | 优先级 | 说明 |
|--------|--------|------|
| **情绪捕捉** | P1 | 通过语音语调分析用户紧张程度，实时影响小人动画（发抖、流汗） |
| **AI扮演多样性** | P1 | 面试官随机人设：温和型、挑剔型、打断型、沉默型 |
| **实时援护增强** | P1 | 陪跑员不只是文字提示，可以语音悄悄话 |
| **面试回放** | P1 | 全程录音，支持回放和AI标注"回答得好的点/需要改进的点" |

### 4.3 内容创新

| 创新点 | 优先级 | 说明 |
|--------|--------|------|
| **真题库** | P1 | 收集各公司真实面试题，按公司/岗位/难度分类 |
| **JD智能分析** | P1 | 粘贴JD自动生成：技能要求解读、面试重点预测、简历关键词建议 |
| **行业黑话词典** | P2 | "弹性工作制"= 加班、"扁平化管理" = 事多钱少 |
| **避坑指南** | P2 | 基于社区数据，标注高风险公司特征 |

### 4.4 商业模式建议

| 模式 | 优先级 | 说明 |
|------|--------|------|
| **免费 + 增值** | P0 | 基础面试免费，高级NPC/无限次面试/简历优化付费 |
| **皮肤商城** | P2 | 像素装扮付费解锁 |
| **企业版** | P2 | 企业定制面试题库、批量评估候选人 |

## 5. 风险与建议

### 5.1 技术风险

| 风险项 | 风险等级 | 应对建议 |
|--------|---------|---------|
| Lip Sync延迟 | 中 | 预加载动画，使用Web Audio API本地分析 |
| AI回答质量不稳定 | 高 | 多轮测试调优Prompt，建立评分标准库 |
| 像素动画生成成本 | 中 | 参考NYX方案，控制单次成本在0.5元内 |
| 语音识别准确率 | 低 | 使用成熟ASR服务（讯飞/阿里/腾讯） |

### 5.2 产品风险

| 风险项 | 风险等级 | 应对建议 |
|--------|---------|---------|
| 用户学习成本高 | 中 | 新手引导、渐进式功能解锁 |
| 游戏化过度影响严肃性 | 中 | 区分"练习模式"和"正式模式" |
| 内容更新压力 | 高 | 建立内容贡献机制（用户投稿面试题） |
| 用户流失点 | 中 | 数据埋点分析流失节点，优化体验 |

### 5.3 合规风险

| 风险项 | 风险等级 | 应对建议 |
|--------|---------|---------|
| 用户隐私（语音数据） | 高 | 明确告知数据用途，提供删除选项，本地处理优先 |
| AI生成内容版权 | 低 | 像素风格侵权风险低，但仍需原创设计 |
| 面试题目版权 | 中 | 标注题目来源，避免直接复制付费题库 |

## 6. 需求确认清单

### 6.1 产品定位相关

- [ ] **目标市场优先级**：国内市场优先还是海外市场？
- [ ] **付费意愿验证**：目标用户的付费价格区间？（建议：月卡9.9元/季卡29.9元）
- [ ] **竞品差异化**：是否接受初期功能与竞品重叠，先快速上线验证？

### 6.2 像素小人相关

- [ ] **生成方式**：MVP阶段是否接受仅问卷生成（无照片）？
- [ ] **动画数量**：初期需要多少种动画状态？（建议：呼吸、说话、聆听、点头、紧张共5种）
- [ ] **装扮系统**：是否需要皮肤商城？（建议V2.0）

### 6.3 面试系统相关

- [ ] **面试类型**：初期支持哪些类型？（建议：行为面、技术面、HR面）
- [ ] **面试时长**：单次面试时长限制？（建议：10-15分钟）
- [ ] **题目来源**：自有题库还是用户投稿？
- [ ] **评分维度**：确认5个维度是否合理？（表达能力/逻辑思维/专业能力/应变能力/情绪管理）

### 6.4 NPC系统相关

- [ ] **NPC优先级**：哪些NPC必须MVP上线？（建议：面试官 + 陪跑员）
- [ ] **NPC交互方式**：纯文字还是语音？（建议：面试官语音，陪跑员可文字提示）
- [ ] **陪跑员触发机制**：自动判断卡壳还是用户主动请求？

### 6.5 数据与隐私相关

- [ ] **语音数据存储**：是否需要存储用户语音？存储多久？
- [ ] **面试记录保留**：保留多久？是否支持用户删除？
- [ ] **第三方服务**：ASR/TTS使用哪家服务商？

### 6.6 技术实现相关

- [ ] **前端框架确认**：React是否确定？是否考虑后续App化？
- [ ] **后端语言确认**：Python是否确定？
- [ ] **AI模型选择**：LLM使用哪家API？（建议：国内用通义千问/文心一言，海外用Claude/GPT）
- [ ] **部署方式**：SaaS还是私有化部署？

### 6.7 商业化相关

- [ ] **收费模式确认**：订阅制还是按次付费？
- [ ] **企业版是否规划**：是否需要企业定制功能？
- [ ] **内容合作**：是否考虑与培训机构/学校合作？
